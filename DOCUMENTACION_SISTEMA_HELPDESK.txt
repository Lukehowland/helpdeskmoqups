================================================================================
                                DESCRIPCIÓN GENERAL
================================================================================

El Sistema Helpdesk es una plataforma completa de gestión de tickets de soporte 
diseñada para permitir a múltiples empresas gestionar su atención al cliente de 
manera profesional y eficiente. La plataforma opera bajo un modelo multi-tenant 
donde cada empresa mantiene sus datos separados y seguros.

CARACTERÍSTICAS PRINCIPALES:
• Sistema multi-empresa con datos separados
• 4 tipos de usuarios con permisos diferenciados
• Gestión completa del ciclo de vida de tickets
• Interfaz moderna y responsiva
• Sistema de roles y permisos granular
• Dashboard personalizado para cada tipo de usuario

================================================================================

================================================================================
                          1. USUARIO FINAL (USER)
================================================================================

DESCRIPCIÓN:
Clientes finales de las empresas que utilizan el sistema para reportar 
problemas, solicitar soporte y dar seguimiento a sus incidencias.

ACCESO AL SISTEMA:
• Email: user@gmail.com (para pruebas)
• URL de acceso: Página principal → Iniciar Sesión

PERMISOS OTORGADOS:
✓ tickets.view.own - Ver solo sus propios tickets
✓ tickets.create - Crear nuevos tickets
✓ tickets.respond.own - Responder a sus propios tickets

FUNCIONALIDADES DISPONIBLES:

A) GESTIÓN DE TICKETS:
   - Crear nuevos tickets de soporte
   - Ver historial completo de sus tickets
   - Responder a tickets existentes
   - Filtrar tickets por estado (abiertos, en progreso, resueltos)
   - Adjuntar archivos e imágenes
   - Categorizar tickets por tipo de problema

B) NAVEGACIÓN PRINCIPAL:
   - Dashboard personal con resumen de tickets
   - Sección "Mis Tickets" con filtros
   - Crear nuevo ticket con botón flotante
   - Centro de ayuda con artículos y FAQs

C) PÁGINAS ESPECÍFICAS:
   /tickets - Vista principal de tickets
   /tickets/create - Formulario de creación de tickets
   /tickets/my-open - Tickets abiertos del usuario
   /tickets/[id] - Detalle específico de un ticket
   /create-ticket - Página dedicada para crear tickets
   /help-center - Centro de ayuda y documentación
   /announcements - Anuncios y novedades
   /profile - Perfil personal del usuario
   /settings - Configuraciones personales

CASOS DE USO PRINCIPALES:

1. REPORTAR PROBLEMA:
   • Usuario identifica un problema
   • Accede al sistema y crea un nuevo ticket
   • Selecciona categoría y prioridad
   • Describe el problema detalladamente
   • Adjunta capturas de pantalla si es necesario
   • Envía el ticket al equipo de soporte

2. DAR SEGUIMIENTO:
   • Usuario revisa el estado de sus tickets
   • Lee respuestas del equipo de soporte
   • Proporciona información adicional
   • Confirma resolución del problema

3. BUSCAR AYUDA:
   • Consulta el centro de ayuda
   • Lee artículos y FAQs
   • Busca soluciones autogestionadas

RESTRICCIONES:
✗ No puede ver tickets de otros usuarios
✗ No puede asignar tickets
✗ No puede cerrar tickets (solo el soporte puede hacerlo)
✗ No tiene acceso a funciones administrativas

================================================================================
                              FLUJO DE TRABAJO GENERAL
================================================================================

1. CREACIÓN DE TICKET:
   Usuario → Crea ticket → Sistema asigna ID → Notifica a agentes

2. ASIGNACIÓN:
   Agente toma ticket → O supervisor asigna → Ticket pasa a "En Progreso"

3. RESOLUCIÓN:
   Agente investiga → Proporciona solución → Usuario confirma → Ticket cerrado

4. ESCALACIÓN:
   Si es complejo → Se asigna a especialista → O se escala a supervisor

5. SUPERVISIÓN:
   Admin de empresa → Monitorea métricas → Optimiza procesos → Reporta

6. ADMINISTRACIÓN:
   Admin de plataforma → Gestiona sistema → Aprueba empresas → Mantiene
